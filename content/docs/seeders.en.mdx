## Seeder Architecture

Swell organizes its seeders in two levels for maximum flexibility:

| Seeder | Usage | Command |
|--------|-------|---------|
| `DatabaseSeeder` | Essential data (roles) | `php artisan db:seed` |
| `DemoSeeder` | Complete demo data | `php artisan db:seed --class=DemoSeeder` |

```bash tab="Base (roles only)"
php artisan db:seed
```

```bash tab="Full demo"
php artisan db:seed --class=DemoSeeder
```

```bash tab="Fresh + Demo"
php artisan migrate:fresh --seed --seeder=DemoSeeder
```

## DatabaseSeeder - Essential Data

The `DatabaseSeeder` only runs the `RoleSeeder` to create the roles required for the application to function.

**Swell** relies on the [spatie/laravel-permission](https://spatie.be/docs/laravel-permission) package for role and permission management.

### Roles Created

- `admin`: full access to Admin pages
- `customer`: users who have placed orders
- `visitor`: basic user

<Callout>
The `RoleSeeder` uses `firstOrCreate()` and can be run multiple times without risk of duplication.
</Callout>

## DemoSeeder - Demo Data

The `DemoSeeder` creates a complete environment to test the application:

### Generated Data

| Seeder | Description |
|--------|-------------|
| `RoleSeeder` | Base roles (admin, customer, visitor) |
| `AdminUserSeeder` | Admin user (`john@doe.com` / `password`) |
| `BannerSeeder` | Promotional banner messages |
| `CatalogSeeder` | Brands, categories and products (fashion theme) |
| `DashboardSeeder` | Users and orders for analytics |

### File Structure

```
database/seeders/
├── DatabaseSeeder.php      # Default entry point (RoleSeeder)
├── DemoSeeder.php          # Demo data entry point
├── RoleSeeder.php          # Base roles
└── Demo/
    ├── AdminUserSeeder.php # Admin user
    ├── BannerSeeder.php    # Banner messages
    ├── CatalogSeeder.php   # Brands, categories, products
    └── DashboardSeeder.php # Dashboard data (users + orders)
```

<Callout title="Admin account">
After running `DemoSeeder`, log in with:
- Email: `john@doe.com`
- Password: `password`
</Callout>

### Product Catalog

The `CatalogSeeder` generates a complete fashion catalog:

**Main categories:**
- Women (Tops, Pants, Dresses & Skirts, etc.)
- Men (T-shirts & Polos, Shirts, Sweats, etc.)
- Accessories (Bags, Belts, Jewelry, etc.)
- Sports (Clothing, Shoes, Equipment)

**Brands:** Zara, H&M, Nike, Adidas, Levi's, Mango, and more.

### Dashboard Analytics

The `DashboardSeeder` generates 6 months of realistic data:
- 30+ users with varied creation dates
- Orders distributed with realistic revenue
- Perfect data for testing admin dashboard charts

## Module Seeders

If modules are enabled, the `DemoSeeder` automatically runs their seeders:

```php
// In DemoSeeder.php
if (config('swell.workspace.enabled', false)) {
    $this->call(WorkspaceModuleSeeder::class);
}
```

## Create a Custom Seeder

To add your own test data:

```bash
php artisan make:seeder CustomSeeder
```

Then add it to `DemoSeeder` or create your own entry point.

## Tips and Best Practices

- **Production**: Only run `php artisan db:seed` (RoleSeeder)
- **Development**: Use `php artisan db:seed --class=DemoSeeder`
- **Testing**: The `DemoSeeder` provides consistent data for manual testing
- **Idempotence**: The `RoleSeeder` can be run multiple times without errors

<Callout>
To go further, consult the Laravel documentation on seeders: [https://laravel.com/docs/seeding](https://laravel.com/docs/seeding)
</Callout>
